Why I'm Not Switching to iOS

February 13, 2023 · 3 min · 457 words

While I have used iOS in the past, I’m currently on GrapheneOS. There are however some things about it I don’t like and that make me consider iOS again. Here are the reasons why I’m not switching.

An iPhone is generally more expensive and I’d argue they are currently too expensive. Maybe they’re also compensating due to inflation, but the prices especially for the iPhone SE are still too high.

My second point does not directly address the iPhone. The whole market moved on but the EU pushes back and it seems like that we can have user replaceable batteries in the future. Therefore I will wait until new phones have replaceable batteries before I upgrade my phone again.

Okay the third point is annoying and it is the main selling point of Android phones for me. I want to put my music on my phone without using iTunes. I exclusively use Linux simply due to the reason that I like the KDE desktop best. I have talked about my reasoning why I don’t use macOS anymore in my article about My Linux Journey So Far. And while I would be fine on Windows I’ve become accustomed to the Linux terminal and Neovim. I would simply feel less efficient on Windows and I would probably be using the WSL most of the times. Putting music on an iPhone using Linux is a huge pain. iTunes does not run on Linux and one has run virtualisation of some kind. And there’s no guarantee that it is going to work in the end. And why? Because Apple makes it hard for us to put our own music on our own devices. Yes, there is VLC. But if I spend a couple hundred on a mobile device I want to put my music on it and use the native mobile player and not some aftermarket solution.

Mobile devices should use USB-C. There is no logical reason for Apple except profit to still use the old lighting connector for iPhones.

So if all of those issues are addressed some day I might switch back to an iPhone. I simply like their UI better than Android. I’m currently using GrapheneOS and its UI is utterly inconsistent. But I don’t like stock Android either. iOS just has a very intuitive and consistent user interface. But as long as iOS is a closed off system I don’t see myself switching to it. I’d at least want to be able to put my own music on it. And it would also be nice to use a browser engine different than WebKit. And no I don’t want additional browser engines. I want to be able to choose which browser engine is used on a system level.
